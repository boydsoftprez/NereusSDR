# NereusSDR v0.3.1 Alpha Test — OpenHPSDR Radios (P1 & P2)

**Hi, and thank you.** You're helping test an **alpha-stage** SDR console. v0.3.0 was pulled within hours of release for a packaging fix — most testers never saw it. v0.3.1 is the first build you'll actually run, and it folds in everything 0.3.0 was supposed to deliver (SSB voice transmit, the rebuilt VPN-reach connection, the expanded Hermes Lite 2 configuration tabs, the status-bar redesign) plus the post-0.3.0 polish: per-profile TX bandwidth control, a user-editable filter preset store, the TX filter overlay on the panadapter, the per-board PA forward-power calibration system, and the **HL2 ATT/filter safety audit closure that clears HL2 SSB transmit for bench testing**. This walkthrough focuses on what's new since v0.2.3.

If you're a returning tester, the **v0.2.3 doc** ([docs/debugging/v0.2.3-alpha-tester-smoketest.md](v0.2.3-alpha-tester-smoketest.md)) is still the right reference for receive-side, hardware tabs, and DSP-grid behavior — those didn't change much. Skim it once for the RX-side test sequence and use this v0.3.1 doc to drive the new test surface.

---

## What's new in v0.3.1 (the short list)

1. **SSB voice transmit works on every supported board, including Hermes Lite 2.** Microphone in, RF out. Full Thetis-faithful audio processing chain (transmit equalizer, leveler, ALC, multi-band compressor, controlled-envelope SSB, phase rotator). 21 factory mic profiles ported verbatim. The HL2 ATT/filter safety audit closed in v0.3.1 — HL2 owners can now bench-TX with the same confidence as the ANAN family.
2. **Per-profile TX bandwidth control.** TxApplet gains TX BW low/high spinboxes and a live status label. Each mic profile now stores its own FilterLow/FilterHigh; debounced WDSP path applies changes without keying glitches. Setup → Transmit → TX Profile gains a TX Filter group for editing.
3. **User-editable filter preset store.** Filter presets are editable per-mode (CW / SSB / AM / FM / DIGI). New Setup → Filters → Filter Presets page with per-mode lists and an edit dialog. Mode-aware preset grid on RxApplet — the buttons reflect the current mode. Shift+click a preset on the TX applet to snap TX BW to the active RX preset.
4. **TX filter overlay on the panadapter and waterfall.** TX bandwidth is drawn on the spectrum and waterfall during MOX/TUNE — orange when transmitting, cyan otherwise. Color pickers for both overlays in Setup → Display → Colors & Theme.
5. **Per-board PA forward-power calibration.** New Setup → PA → Watt Meter page hosts a 10-cal-point grid per board class (None / Anan10 / Anan100 / Anan8000; HL2 maps to Anan10). The `alex_fwd` reading is now routed through `CalibratedPAPower` interpolation. SWR protection scaling factor applied at every `setTxDrive` site. Setup → PA → PA Values shows 9 live telemetry fields.
6. **Connection workflow rebuilt.** Radios reachable through a VPN tunnel (WireGuard / ZeroTier / Tailscale) now connect. New 16-model picker. Auto-connect-on-launch. Spectrum fades to a "DISCONNECTED" overlay when the link drops.
7. **Hermes Lite 2 configuration surface expanded.** New Hermes Lite Options tab, I2C control, I/O pin state, and N2ADR HERCULES toggle that writes all 13 SWL bands × 7 pins.
8. **Status-bar redesign.** New compact title-bar connection segment, responsive receive-info badges that drop in priority as the window narrows, ADC overload badge, station-name anchor.
9. **PA voltage on MkII boards reads correctly now** — was off by ~20 % in earlier builds.
10. **Setup IA reshape.** New top-level **PA** category (gated on `caps.hasPaProfile`) with PA Gain / Watt Meter / PA Values sub-pages. Hardware → Calibration tab is always visible.
11. **macOS builds are signed and notarized.** Gatekeeper accepts them without right-click → Open. Windows portable bundle ships with the MinGW runtime DLLs that were missing from v0.3.0.

## Critical safety notes — read before you key up

**TX bench testing requires a dummy load.** The DSP chain is still relatively new code. Don't transmit into your antenna without verifying SWR and forward power on a dummy load first.

**Per-radio TX status:**

| Radio | TX safe to bench? | Notes |
|---|---|---|
| ANAN-G2 / Saturn | **Yes** — primary TX dev target | Bench-tested by JJ |
| ANAN-7000DLE / 8000DLE | **Yes — likely fine** (same P2 path) | Bench reports welcome |
| ANAN-200D / 100D / 100B / 100 / 10 / 10E | **Yes — likely fine** (P1 TX path verified) | Watch ADC overload badge carefully |
| Hermes / Hermes II / Angelia / Orion / Orion MkII | **Yes — likely fine** (P1 TX path verified) | Bench reports welcome |
| **Hermes Lite 2** | **Yes — bench-cleared in v0.3.1** | ATT/filter safety audit closed; the step-attenuator and filter gating were verified end-to-end on HL2 hardware. Run the HL2 TX pre-flight checklist below before keying up — confirm the step-attenuator reads in the expected −28..+32 dB signed range, the N2ADR / SWL filter selection matches your band, and your PA cal-point row for Anan10 (which HL2 maps to) has been verified on the Watt Meter page. |

**Other safety items:**

- The new ADC Overload badge (yellow on any clip, red on level >3) is your friend — if it lights red on TX, you're overdriving the front end. Reduce drive.
- TUNE button on the TX applet kicks a single-tone carrier at the configured tune drive. Use it on a dummy load first to confirm forward power and SWR readouts before keying voice.
- VOX is on by default for SSB. If you don't want VOX, toggle it off in the TX applet before plugging your mic in — otherwise ambient noise can MOX the radio.
- The TX bandwidth overlay on the panadapter/waterfall (orange in TX, cyan in RX) should match what you expect for the active mic profile. If it doesn't, check Setup → Transmit → TX Profile → TX Filter for the per-profile bandwidth.

**HL2 TX pre-flight checklist (new in v0.3.1):**

1. Setup → Hardware → Hermes Lite Options → step-attenuator spinbox accepts the signed −28..+32 dB range. Confirm your value sits in the range you expect (negative = no attenuation; the chip drops below 0 dB into the gain side as a Cortex-style trim).
2. Setup → Hardware → Hermes Lite Control → SWL → 13 SWL bands × 7 pins matrix matches the filter board you're using. If you have an N2ADR board, the HERCULES toggle writes all 13 pin-7 entries in one go.
3. Setup → PA → Watt Meter → Anan10 cal-point row reflects the cal-point values the HL2 was bench-verified against (or your own bench cal). The HL2 maps to Anan10 board class.
4. Dummy load attached. Drive at 10–25 % for the first key-up.
5. Watch the ADC overload badge during TX — yellow is OK, red means reduce drive.

## Before you start

### Things you need

- A supported OpenHPSDR radio on your network
- The platform-appropriate v0.3.1 build from <https://github.com/boydsoftprez/NereusSDR/releases/tag/v0.3.1>
- A dummy load if you're going to test TX (50 Ω, rated for your radio's full power output)
- A mic (any common ham mic into the radio's mic input, or a USB mic going through the PC mic source if you're testing the Pc/Composite paths)
- A few minutes of free time on a quiet band

### Get the build

**macOS Apple Silicon (M-series Macs):**

Download `NereusSDR-0.3.1-macOS-apple-silicon.dmg`. Open the DMG, drag NereusSDR.app to Applications, eject the DMG, launch from Applications. **Gatekeeper should accept it on first launch** — the build is now Apple Developer ID-signed and notarized.

If Gatekeeper still blocks (rare; happens on a stale system policy cache), right-click the app → Open → Open. After the first successful launch, Gatekeeper remembers and stops asking.

If you want to install the .pkg variant (system-wide install with the CoreAudio HAL plugin pre-installed for VAX), download `NereusSDR-0.3.1-macOS.pkg` and double-click. The PKG is also notarized.

**Linux (x86_64 desktop):**

Download `NereusSDR-0.3.1-x86_64.AppImage`, `chmod +x` it, run it.

```
chmod +x NereusSDR-0.3.1-x86_64.AppImage
./NereusSDR-0.3.1-x86_64.AppImage
```

If you get a FUSE error, install `libfuse2` (Ubuntu/Debian) or use `--appimage-extract-and-run`.

**Linux (aarch64 / ARM64 — Raspberry Pi 5, etc.):**

Same drill with `NereusSDR-0.3.1-aarch64.AppImage`. GPU spectrum is currently disabled on this build — you'll get the CPU-rendered fallback.

**Windows (x64):**

Two options:

1. **Installer:** `NereusSDR-0.3.1-Windows-x64-setup.exe`. Double-click. SmartScreen may warn — click **More info → Run anyway**. The installer is unsigned (no Authenticode certificate yet for the alpha line). Installs to `Program Files` and creates Start Menu / desktop shortcuts.
2. **Portable:** `NereusSDR-0.3.1-Windows-x64-portable.zip`. Unzip anywhere, run `NereusSDR.exe`. No installer, no shortcuts, just the binary + DLLs in a folder you can move around.

Both Windows builds bundle the Visual C++ 2022 runtime and use the Qt 6.8 LTS framework.

### Verify the signature (optional but recommended)

Every artifact ships with a detached `.asc` GPG signature, plus a `SHA256SUMS.txt` with all checksums in one file (also signed as `SHA256SUMS.txt.asc`).

To verify, import the maintainer key:

```
gpg --recv-keys 20C284473F97D2B3
gpg --verify SHA256SUMS.txt.asc SHA256SUMS.txt
sha256sum -c SHA256SUMS.txt   # Linux / macOS
```

The signature should report `Good signature from "JJ Boyd <kg4vcf@gmail.com>"`.

## The actual test

### 1. Launch and look around

Open the app. You should see:

- The **title bar** has a new compact connection segment up top: a state dot (gray when offline, green when connected), and slots for transmit Mbps, ping ms, receive Mbps, and a music-note audio indicator. Hover the segment for a multi-line tooltip.
- The **status bar** at the bottom is redesigned. Center has a clickable **STATION** block with "Click to connect" in dashed-red when offline. To the left of STATION is a row of badges that will light up once you connect (mode, filter, AGC, NR, NB, APF, SQL). To the right is a **MetricLabel strip** with CAT / TCI / PA voltage / CPU / PA-OK / TX status.
- The **Connection Panel** auto-opens on first launch. If it didn't (e.g., you've launched this build before), open it via Radio menu → Connect (⌘K on macOS, Ctrl+K elsewhere).

### 2. Discover and connect — the new path

The connection workflow has been rebuilt. Try it both ways:

**Path A — broadcast discovery (the classic):**

1. In the Connection Panel, click **↻ Scan** in the table header.
2. Within ~2 seconds you should see your radio appear in the list with a 🟢 Online state pill, "Last Seen" populated to "just now," and your radio's name + protocol + IP.
3. Click the row, click **Connect**. The state dot in the title bar should turn green. The spectrum should start moving, audio should come through if you're tuned to a busy frequency.

**Path B — unicast probe (new — this is what makes VPN reach work):**

This is the path that lets you reach a radio that *isn't* on your local broadcast domain — e.g., an HL2 at home you've tunneled to via WireGuard, ZeroTier, or Tailscale.

1. In the Connection Panel, click **+ Add radio**.
2. The Add Radio dialog now has a **16-model picker organized by silicon family** (Auto-detect, Atlas, Hermes (3 boards), Hermes II (2), Angelia, Orion, Orion MkII (5), Hermes Lite 2, Saturn (2)). Pick your model — or leave on Auto-detect to let the probe figure it out.
3. Enter the IP address and port (default 1024 for OpenHPSDR discovery).
4. Click **Probe and connect now**. The dialog opens a 1.5 s probe — parallel P1 + P2 attempts — and either:
   - **Success** → dialog auto-closes, the row lands in your radio table tagged "(probe)", and the connection proceeds.
   - **Failure** → dialog stays open with form preserved, a red error band describes what happened (no reply, wrong protocol version, etc.).

If you have a radio reachable only through a tunnel, **this is one of the high-value tests carried forward from v0.3.0** — most testers never got to run it because v0.3.0 was pulled within hours. Confirm it works in your tunnel topology and let us know if not.

5. Click **Save offline** instead if you just want to register the radio without connecting yet — it'll stay in the table and you can connect later.

### 3. Auto-connect on launch

1. Once you've successfully connected to a radio, open the detail pane in the Connection Panel and check **Auto-connect on launch**.
2. Quit the app cleanly (⌘Q / Ctrl+Q).
3. Relaunch. The app should auto-connect to that radio without you opening the Connection Panel.
4. If auto-connect fails (radio off, network changed), the panel auto-opens with the target highlighted offline and a status-bar message — no silent failure.

### 4. Disconnect overlay

1. Connected? Good.
2. Power-cycle your radio, or pull its Ethernet, or break the tunnel.
3. Within ~5 seconds the spectrum should fade to ~40 % opacity with a "DISCONNECTED" label and a "Click to open connection panel" hint.
4. Click anywhere on the faded spectrum → Connection Panel opens.
5. The radio's row should switch to 🔴 Offline (or 🟡 Stale, depending on timing). Saved entries stay; if it was a discover-only entry, it ages out after 60 seconds.

This closes a v0.2.x complaint where a dropped radio just left a frozen spectrum with no feedback.

### 5. RX walkthrough — quick check

The RX path is unchanged from v0.2.3. **Refer to the v0.2.3 doc steps 4-15** for the full RX exercise. At minimum on this build:

1. Tune to a busy SSB frequency.
2. Listen — audio should be clean, smooth, with no stutter.
3. Try the noise reduction stack (NR1/2/3/4 + DFNR + MNR — see v0.2.3 doc step 13).
4. Check the dBm scale strip on the right edge of the spectrum (v0.2.3 step 12).
5. Verify Alex antenna switching follows band changes (v0.2.3 step 14).

If anything in the RX behavior is *different* from v0.2.3 — not just buggy, but actually behaving differently than it did in 0.2.3 — that's a regression we want to hear about.

### 6. Transmit test — the big new one

**Read the safety notes at the top of this doc again before keying up.** Dummy load, sober power output, watch the ADC overload badge.

#### 6.1 TUNE — single-tone carrier first

1. Connect a dummy load to your radio's antenna port.
2. Set drive level to a low value (10-25 % is plenty for the dummy load test). The TX applet has a Drive slider.
3. On the TX applet, click **TUNE**. The radio should key up with a single-tone carrier.
4. Watch the title bar's TX indicator and the STATION badge. The transmit Mbps in the title-bar segment should jump.
5. Confirm forward power on your dummy load's wattmeter (if it has one) is reasonable for the drive level.
6. Click TUNE again to unkey. The radio should drop out of TX cleanly.

If TUNE works, your basic TX wire path is alive.

#### 6.2 SSB voice — the real test

1. Plug your mic into the radio's mic input (or use the Pc/Composite mic source if you've set up VAX).
2. Set mic gain on the TX applet to a sensible starting point (try 30-50 %).
3. Make sure VOX is **off** for now — toggle it on the TX applet if it's on. We want push-to-talk control during testing.
4. Hold the radio's PTT (or press MOX in the app) and speak.
5. You should see:
   - The S-meter on the receive side go quiet (because RX is muted)
   - The TX-side level meters move (mic level, ALC if armed)
   - The transmit Mbps in the title bar jump
   - Forward power on your dummy load
6. Listen on a second receiver if you have one available. The audio should be intelligible SSB voice.

#### 6.3 Speech processor — TX EQ / Leveler / ALC

This is the new processing chain. Open the **TX applet**:

1. Find the toggle row `[LEV] [EQ] [PROC]`. These switch on/off the leveler, transmit EQ, and the multi-band compressor (CFC) respectively.
2. Click **EQ** — the EQ enables. Right-click EQ → **Open EQ Editor…** (or use Tools menu) to open the parametric EQ dialog.
3. The EQ editor is a 10-band parametric with:
   - A drag-and-click parametric panel showing the curve
   - 10 sliders for the gain at each band
   - Frequency spinboxes per band
   - Preamp slider for overall level
   - Profile dropdown — try a few of the **21 factory profiles** (Heil PR40, Heil PR781, Yamaha CM500, Behringer XM8500, etc.) ported verbatim from Thetis
   - Save / Save As / Delete buttons for your own profiles
   - Reset button (now actually resets — the v0.2.3-era reset bug is fixed)
4. Try a Heil profile, transmit again, listen on the second receiver. Voice should sound noticeably more processed (broadcast-y) compared to flat.
5. Click **PROC** to enable the multi-band compressor (Continuous Frequency Compressor — CFC). Right-click PROC → Open CFC Dialog. The CFC dialog has:
   - Profile dropdown
   - Global enable + post-EQ gain
   - 30 spinboxes for per-band F / COMP / POST-EQ across 10 bands
   - Phase Rotator and CESSB toggles in their own group boxes
6. Pick a profile, transmit, listen for the change.

#### 6.4 Two-tone IMD test (optional, but useful for measuring transmitter linearity)

1. On the TX applet, click the **2-TONE** button to enable two-tone test mode.
2. Configure the two-tone settings on Setup → Test → Two-Tone (frequencies, levels, drive source).
3. MOX on. The radio transmits two tones — you should be able to measure third-order IMD with a spectrum analyzer.
4. The two-tone mode bypasses the mic path — you don't need to be plugged in.

#### 6.5 Verify PA voltage reads correctly

Important regression-watch item: the PA voltage formula was off by ~20 % in v0.2.x.

1. Connected to a MkII-class radio (Saturn / G2 / 8000D / 7000DLE / OrionMkII / Anvelina Pro 3)?
2. Look at the **PA voltage label** in the right-side strip of the status bar.
3. On a typical 13.8 V supply, it should read **~13.3-13.8 V** (depends on your exact PSU and load). Any number around 11.0 V means the fix didn't take and we have a regression.

Bench-verified by JJ on an ANAN-G2: reads 13.3 V.

#### 6.6 Per-profile TX bandwidth (new in v0.3.1)

1. On the **TX applet**, find the TX BW low/high spinboxes and the live status label (e.g. `100 – 3100 Hz · 3.0 kHz`).
2. Adjust the high spinbox up/down. Watch the status label refresh and the panadapter overlay width change as the WDSP filter follows.
3. Open Setup → Transmit → **TX Profile** → TX Filter group. The same FilterLow/FilterHigh values should appear here, scoped to the active mic profile.
4. Save the profile, switch to a different one, switch back. The bandwidth should restore.
5. Shift+click a filter preset on the TX applet to **snap TX BW to the active RX preset**.

#### 6.7 User-editable filter preset store (new in v0.3.1)

1. Setup → Filters → **Filter Presets**. Pick a mode (CW / SSB / AM / FM / DIGI). The per-mode preset list should populate.
2. Click a preset → **Edit**. Adjust low cut, high cut, name. Save.
3. Close Setup. Open RxApplet. The mode-aware preset grid should reflect your edits for the current mode.
4. Switch modes. Confirm the preset grid swaps to the right mode's list.
5. Restart the app. Your edits should persist.

#### 6.8 TX filter overlay on the panadapter and waterfall (new in v0.3.1)

1. While receiving (no MOX), confirm the receive filter overlay shows in **cyan** at the active RX bandwidth.
2. Press TUNE or MOX. The TX filter overlay should appear in **orange** at the TX bandwidth.
3. Both overlays should sit **under** the bandplan and frequency labels.
4. Setup → Display → **Colors & Theme**. Use the TX/RX overlay color pickers to change the colors.
5. Restart the app. The colors and the overlay visibility should persist.

#### 6.9 PA forward-power calibration (new in v0.3.1)

1. Setup → **PA** (top-level Setup category, gated on `caps.hasPaProfile`).
2. **Watt Meter** sub-page. The 10 cal-point grid for your board class should populate.
3. Edit a cal-point spinbox. Confirm it saves (per-MAC).
4. **PA Values** sub-page. 9 telemetry fields should populate live: FWD / REV / SWR / PA current / temperature / supply volts / ADC overload / raw FWD-ADC / raw REV-ADC. Note this only updates while the radio is connected.

### 7. Hermes Lite 2 deep checks (HL2 owners only)

**HL2 SSB transmit is bench-cleared in v0.3.1.** Run the HL2 TX pre-flight checklist in the safety section above before keying up. This section walks both the RX/configuration surface and the new TX path.

#### 7a. Configuration surface

1. Connect to your HL2.
2. Setup → Hardware → **Hermes Lite Options** — verify:
   - I2C control widgets show real I2C transactions
   - I/O Pin State table updates live
3. Setup → Hardware → **OC Outputs** (renamed to "Hermes Lite Control" when HL2 is connected) → SWL sub-tab — should show the **13 SWL bands × 7 pins matrix**. Toggle a few cells, verify they persist per-MAC across an app restart.
4. **N2ADR HERCULES toggle** — there should be a toggle that, when on, writes all 13 SWL pin-7 entries. Mirrors the HERCULES behavior in mi0bot's HL2 fork.
5. **Step attenuator** — the spinbox should accept the **signed −28..+32 dB range** (was −28..+0 in earlier builds). Try negative values.
6. Disconnect, reconnect — settings should persist per-MAC, not leak between radios if you have multiple HL2s.

#### 7b. PA cal (new in v0.3.1)

1. Setup → PA → **Watt Meter**. The HL2 maps to the **Anan10** board class — confirm the cal-point grid for Anan10 reflects values that make sense for your setup, or run your own bench cal.
2. Setup → PA → **PA Values**. The 9 telemetry fields should populate live: FWD / REV / SWR / PA current / temperature / supply volts / ADC overload / raw FWD-ADC / raw REV-ADC.

#### 7c. HL2 TX bench (new in v0.3.1)

The HL2 ATT/filter safety audit closed during v0.3.1 polish. Bench TX is now cleared, but use the same dummy-load discipline as ANAN testing.

1. Dummy load attached. Drive at 10–25 % for the first key-up.
2. **TUNE** — click TUNE on the TX applet. The HL2 should key up with a single-tone carrier; watch forward power on the dummy load wattmeter. Click TUNE again to unkey.
3. **SSB voice** — same flow as section 6.2 but on your HL2. Watch the ADC overload badge during TX.
4. **TX bandwidth on the panadapter** — confirm the orange TX overlay appears during MOX with a width matching your active mic profile's FilterLow/FilterHigh.
5. If anything looks off — relay clicking, wrong forward power, ADC overload red — drop drive and report.

### 8. Status-bar redesign — adapt to window width

1. Maximize the window. The receive-info badges should show all 7 (mode, filter, AGC, NR, NB, APF, SQL) side-by-side in 3 BadgePairs plus SQL.
2. Slowly narrow the window. As it gets narrower, the badge pairs should:
   - First stack vertically (medium width)
   - Then drop in priority order (narrowest) — but **mode and filter never drop**
3. Dropped badges show up in an **OverflowChip ("…")** with a hover tooltip listing what was hidden.
4. Try the 30 px hysteresis deadband — the boundary between layouts should not flash back and forth as you drag near the threshold.

### 9. CPU readout toggle

1. Right-click the **CPU MetricLabel** in the right-side status strip.
2. A context menu offers **System CPU** / **App CPU**. Switch between them.
3. The reading and the label should change. Persist across app restart.
4. Default is System CPU (matches Thetis behavior).

### 10. ADC Overload badge

1. Connected? Tune to a band with strong signals (40 m at night, 80 m night, etc.).
2. Disable the preamp / step attenuator to deliberately push the ADC.
3. The **ADC Overload badge** should appear (yellow on any level > 0, red on level > 3) between PA-OK and TX in the right strip.
4. The badge auto-clears after 2 seconds of no overload.
5. Hover for a tooltip.

### 11. Network Diagnostics

1. Help menu → Network Diagnostics, or right-click the title-bar connection segment → Network Diagnostics.
2. The dialog should show 4 sections of health data.
3. **Watch the ping (RTT)** — should show your actual round-trip in ms. On a LAN it should read sub-millisecond accurately (the new min-filter window fixes a v0.2.3 bug where mean smoothing showed garbage on sub-ms LAN).
4. **TX / RX rate** should show real Mbps (was 0.00 Mbps in earlier builds — that bug is fixed).
5. Jitter / packet loss / packet gap rows show em-dash with a "Not yet measured" tooltip. That's intentional — proper measurement instrumentation is later work.

### 12. Quit and relaunch — persistence check

1. Quit cleanly (⌘Q / Ctrl+Q).
2. Relaunch.
3. Verify everything persists:
   - Last connected radio
   - VFO frequency / mode / filter
   - Mic profile selection
   - TX EQ / Leveler / ALC / CFC settings
   - Auto-connect-on-launch flag
   - Container layout
   - Spectrum / waterfall settings
   - HL2 N2ADR + step attenuator + SWL band selections (per-MAC)

If anything resets on you that shouldn't — that's a bug.

- Most of Setup → Transmit (apart from TX EQ / Leveler / ALC / CFC pages which DO work)

- Setup → Keyboard
- Setup → CAT / Network

## Things that might look weird but aren't bugs

- **Spectrum / waterfall ship defaults are 12 dB lower than v0.2.3.** Intentional — band noise no longer floods the panadapter floor on a typical residential antenna. If you preferred the old look, Setup → Display → Reset to legacy.
- **Clarity defaults to ON for new installs.** Adaptive auto-tune is the better first-launch experience than asking users to find the toggle.
- **PSU widget is gone.** Source-first audit confirmed the underlying ADC channel is dead data in Thetis — no callers. PA drain voltage label remains as the sole supply readout.
- **OC Outputs page is renamed to "Hermes Lite Control" when an HL2 is connected.** It's the same page, with HL2-specific sub-tabs added.
- **Notarization on macOS can take a while at first launch.** Apple's Gatekeeper does an online ticket check the first time. If it sits for 10-20 seconds, that's normal.

- **VPN/tunnel reach.** The unicast probe is the headline change. If you have an HL2 at home behind WireGuard/ZeroTier/Tailscale and a remote NereusSDR client, this is the test we most want feedback on.
